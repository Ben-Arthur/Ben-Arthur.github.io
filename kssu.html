Collision-box data structure

The game allocates a linked list of "objects" (starting at memory address 0x02076f00) and stores pointers to various properties it needs, so that it can communicate what happens to another object when they intersect. The second pointer of each object is the collision box data:

Offset Description
0x00 Padding? (0x0000)
0x02 Hitbox frame duration (behavior is generally more complicated than this)
0x03 Flag for end(01)/repeat(04)/continue(00)? once timer reaches 0
0x04 Hitbox Damage (signed)
0x06 unknown
0x07 unknown
0x08 unknown
0x09 unknown
0x0A Effect (normal, can't hit bosses, fire, ice, electric, etc.)
0x0B Strength (related to length/direction of knockback animation)
0x0C unknown
0x0D unknown
0x0E On-hit sound effect
0x10 Offset of the collision-box's center point from Kirby's position (x,y)
0x14 Offset of the top left corner from the center point (x,y)
0x18 Offset of the bottom right corner from the top left corner (x,y)
